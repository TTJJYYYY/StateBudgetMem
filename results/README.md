# Results Directory Guide

This directory contains machine-readable outputs from offline baselines,
versioning runs, view experiments, staleness analysis, and defense demos.

Do not treat every timestamped file as a paper-ready result. A result becomes a
formal result only when its command, config, dataset, seed, top-k, budget, git
commit if available, and limitations are recorded.

## Formal Results

Formal results are reproducible experiment outputs that can be cited in reports
after review.

| Path | Produced by | Contents |
|---|---|---|
| `raw/` | `statebudgetmem run --config configs/baseline.yaml` | Per-query TF-IDF retrieval rows. |
| `summaries/` | `statebudgetmem run --config configs/baseline.yaml` | TF-IDF summary JSON/CSV files. |
| `views/raw/` | `python tools/views/run_views_experiment.py` | Per-query flat/current/dual view retrieval rows. |
| `views/summaries/` | `python tools/views/run_views_experiment.py` | Flat/current/dual comparison summaries. |
| `versioning/memora/` | `python tools/versioning/run_memora_versioning.py` | Versioning transition summaries and reports. |
| `baselines/` | `python -m statebudgetmem.baselines.runner` | Baseline-family comparison outputs. |

Before using these numbers in slides or text, verify the matching raw file and
summary file were generated by the same run ID.

## Temporary Results

Temporary results are useful for smoke tests, demos, or debugging, but should
not be cited as final experiment evidence without rerunning and recording the
full setup.

| Path | Meaning |
|---|---|
| `defense_demo/` | Combined demo summaries from `python tools/demo/run_defense_demo.py`. Use `latest_summary.json` for the most recent local demo. |
| `staleness/` | Offline stale-retrieval checks from `statebudgetmem analyze-staleness --backend tfidf`. Useful diagnostics; promote only after recording the exact command and dataset. |
| Repeated timestamped files | Local reruns. Keep them if needed for traceability; do not cherry-pick only the best run. |

## Recommended Demo Commands

Run these from the repository root after installing the project with
`python -m pip install -e ".[test]"`.

| Purpose | Command | Expected output |
|---|---|---|
| Full defense demo | `python tools/demo/run_defense_demo.py` | `results/defense_demo/latest_summary.json` plus component outputs. |
| TF-IDF baseline | `statebudgetmem run --config configs/baseline.yaml` | `results/raw/<run_id>.jsonl`, `results/summaries/<run_id>.json`, and `results/summaries/<run_id>.csv`. |
| CLI availability | `python -m statebudgetmem.cli --help` | CLI subcommand list. |

## Optional Commands

| Purpose | Command | Output |
|---|---|---|
| Views only | `python tools/views/run_views_experiment.py` | `results/views/raw/*.jsonl` and `results/views/summaries/*.json|csv`. |
| TF-IDF stale analysis | `statebudgetmem analyze-staleness --backend tfidf` | `results/staleness/raw/*.jsonl` and `results/staleness/summaries/*.json|csv`. |
| MemoryBank evaluation | `statebudgetmem evaluate-memorybank --output results/memorybank/evaluation.json` | MemoryBank comparison JSON. |
| MemoryBank demo UI | `statebudgetmem-demo` | Starts the Gradio demo from `statebudgetmem.baselines.memorybank.demo`. |

Some tool scripts import `statebudgetmem` directly. If the package is not
installed, run them with `PYTHONPATH=src` or install the project first.

## Result Hygiene

- Do not manually edit metrics to match expectations.
- Do not delete raw files that support reported summaries.
- Do not report terminal-only output as an experiment result.
- Keep failed-run logs when a failed experiment matters for the research record.
- Use `defense_demo/latest_summary.json` for quick meetings, and rerun formal
  experiments before writing final claims.
